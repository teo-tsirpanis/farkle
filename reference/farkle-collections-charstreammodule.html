Farkle

CharStream

Namespace: Farkle.Collections

Functions to create and work with CharStreams. They are not thread-safe.

Functions and values

Function or valueDescription
consume doUnpin cs arg3
Signature: doUnpin:bool -> cs:CharStream -> CharSpan -> unit

Advances a CharStream's position by as many characters the given CharSpan indicates. These characters will not be shown again on new CharStreamViews. Calling this function does not affect the pinned CharSpans. Optionally, the characters before the span can be marked to be released from memory.
consumeOne doUnpin cs
Signature: doUnpin:bool -> cs:CharStream -> unit

Advances a CharStream's position by one character. These characters will not be shown again on new CharStreamViews. Calling this function does not affect the pinned CharSpans. Optionally, this character and all before it can be marked to be released from memory.
extendSpanByOne cs arg2
Signature: cs:CharStream -> CharSpan -> CharSpan

Creates a new CharSpan that spans one character more than the given one. A CharStream must also be given to validate its length so that out-of-bounds errors are prevented.
extendSpans arg1 arg2
Signature: CharSpan -> CharSpan -> CharSpan

Creates a new CharSpan from two continuous spans, i.e. that starts at the first's start, and ends at the second's end. Returns None if they were not continuous.
getCurrentIndex(cs)
Signature: cs:CharStream -> CharStreamIndex

Creates a CharStreamIndex from a CharStream that points to its current position.
ofReadOnlyMemory(mem)
Signature: mem:(type) -> CharStream

Creates a CharStream from a ReadOnlyMemory of characters.
ofString(x)
Signature: x:string -> CharStream

Creates a CharStream from a string.
ofTextReader(textReader)
Signature: textReader:TextReader -> CharStream

Creates a CharStream from a TextReader. It must be disposed afterwards.
ofTextReaderEx bufferSize textReader
Signature: bufferSize:int -> textReader:TextReader -> CharStream

Creates a CharStream that lazily reads from a TextReader. The size of the stream's internal character buffer is specified. This buffer holds the characters that are the data for a terminal under discovery. If a terminal is longer than the buffer's size, the buffer becomes twice as long each time. The default buffer size is 256 characters. If the specified size is not positive, the default is used. Also, the character stream must be disposed afterwards.
pinSpan arg1 arg2
Signature: CharStream -> CharStreamIndex -> CharSpan

Creates a CharSpan that contains the next idxTo characters of a CharStream from its position. On the dynamic block character stream, it ensures that the characters in the span stay in memory.
readChar cs c idx
Signature: cs:CharStream -> c:outref<char> -> idx:byref<CharStreamIndex> -> bool

Reads the idxth character of cs, places it in c and returns true, if there are more characters left to be read. Otherwise, returns false.
unpinSpanAndGenerate(...)
Signature: symbol:'symbol -> fPostProcess:CharStreamCallback<'symbol> -> cs:CharStream -> CharSpan -> obj
Type parameters: 'symbol

Creates an arbitrary object out of the characters at the given CharSpan. After that call, the characters at and before the span might be freed from memory, so this function must not be used twice.
unpinSpanAndGenerateString
Signature: CharStream -> CharSpan -> string

Creates a string out of the characters at the given CharSpan. After that call, the characters at and before the span might be freed from memory, so this function must not be used twice. It is recommended to use the unpinSpanAndGenerate function to avoid excessive allocations, unless you specifically want a string.
