Farkle

CharStream

Namespace: Farkle.IO

Functions to create and work with CharStreams. They are not thread-safe.

Functions and values

Function or valueDescription
advance doUnpin cs idxTo
Signature: doUnpin:bool -> cs:CharStream -> idxTo:uint64 -> unit
Attributes:
[<CompiledName("Advance")>]

Advances a CharStream's position just after the given index. Further reads should start one character after it. Calling this function does not affect the pinned CharSpans. Optionally, the characters before the index can be marked to be released from memory.

CompiledName: Advance
advanceByOne doUnpin cs
Signature: doUnpin:bool -> cs:CharStream -> unit
Attributes:
[<CompiledName("AdvanceByOne")>]

Advances a CharStream's position by one character. Further reads should start from the next one. Calling this function does not affect the pinned CharSpans. Optionally, this character and all before it can be marked to be released from memory.

CompiledName: AdvanceByOne
extendSpan cs span endIdx
Signature: cs:CharStream -> span:CharSpan -> endIdx:uint64 -> CharSpan
Attributes:
[<CompiledName("ExtendSpan")>]

Updates the ending index of a CharSpan. It must not exceed the stream's number of currently read characters.

CompiledName: ExtendSpan
extendSpanByOne cs span
Signature: cs:CharStream -> span:CharSpan -> CharSpan
Attributes:
[<CompiledName("ExtendSpanByOne")>]

Creates a new CharSpan that spans one character more than the given one. A CharStream must also be given to validate its length so that out-of-bounds errors are prevented.

CompiledName: ExtendSpanByOne
getPositionAtIndex cs idx
Signature: cs:CharStream -> idx:uint64 -> Position
Attributes:
[<CompiledName("GetPositionAtIndex")>]

Returns the position of the character at idx. It cannot be less than the stream's index of the current position.

Creates a CharStream that lazily reads from a TextReader. The size of the stream's internal character buffer is specified. Also, the character stream can be disposed if the reader is no more needed.
pinSpan cs idxTo
Signature: cs:CharStream -> idxTo:uint64 -> CharSpan
Attributes:
[<CompiledName("PinSpan")>]

Creates a CharSpan that contains the next idxTo characters of a CharStream from its position. You must call this function before calling advance. Despite the name, nothing happens if you don't unpin it.

CompiledName: PinSpan
readChar cs idx c
Signature: cs:CharStream -> idx:uint64 -> c:outref<char> -> bool
Attributes:
[<CompiledName("Read")>]

Reads the idxth character of cs, places it in c and returns false only if input ended. Only characters in memory are allowed to be read or one character past the length so far.

CompiledName: Read
unpinSpanAndGenerate(...)
Signature: symbol:'symbol -> transformer:ITransformer<'symbol> -> cs:CharStream -> CharSpan -> obj
Type parameters: 'symbol Attributes:
[<CompiledName("UnpinSpanAndGenerate")>]

Creates an arbitrary object out of the characters at the given CharSpan. After that call, the characters at and before the span might be freed from memory, so this function must not be used twice.

CompiledName: UnpinSpanAndGenerate
unpinSpanAndGenerateString cs c_span
Signature: cs:CharStream -> c_span:CharSpan -> string
Attributes:
[<CompiledName("UnpinSpanAndGenerateString")>]

Creates a string out of the characters at the given CharSpan. After that call, the characters at and before the span might be freed from memory, so this function must not be used twice. It is recommended to use the unpinSpanAndGenerate function to avoid excessive allocations, unless you specifically want a string.

CompiledName: UnpinSpanAndGenerateString
