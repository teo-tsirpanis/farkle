Farkle.Grammar Namespace

TypeDescription
AdvanceMode

A type indicating how a group advances.
DFAEdges

The edges of a DFA state, that matche a character to the next state, using a custom data structure.
DFAState

A DFA state. It defines the logic that produces tokens out of strings. It consists of edges that the tokenizer follows, depending on the character it encounters.
DFASymbol

A symbol that can be yielded by the DFA.
EndingMode

A type indicating how the ending symbol of a group is handled.
Group

A lexical group. In GOLD, lexical groups are used for situations where a number of recognized tokens should be organized into a single "group". This mechanism is most commonly used to handle line and block comments. However, it is not limited to "noise", but can be used for any content.
GroupEnd

A symbol signifying the end of a group.
GroupStart

A symbol signifying the start of a group.
LALRAction

An action to be taken by the LALR parser according to the given Terminal.
LALRState

A LALR state. Many of them define the parsing logic of a Grammar.
Noise

A symbol which is produced through a DFA, but is not significant for the grammar and is discarded. An example of a noise symbol would be a source code comment.
Nonterminal

A symbol which is produced by a concatenation of other Terminals and Nonterminals, as the LALR parser dictates.
Production

A sequence of Terminals and Nonterminals that can produce a specific Nonterminal.
Symbols

A type containing all symbols of a grammar, grouped by kind. Group start and end symbols can be found at the group table.
Terminal

A symbol which is produced through a DFA, and is significant for the grammar.

Farkle.Parser Namespace

TypeDescription
ExpectedSymbol

A symbol that was expected at the location of a syntax error.
InternalError

An internal error. These errors are known errors a parser might experience. An encounter of it is most certainly a library bug (or deliberately corrupted grammars). The user is encouraged to report it to GitHub. The ToString() method is not overriden here, because users don't need to know that many details about them.
Message<'a>

A log message that contains a position it was encountered.
ParseError

An exception to be thrown when parsing goes wrong. It is thrown by the Parser and Tokenizer APIs, and caught by the RuntimeFarkle.
ParseErrorType

An error the parser encountered.
ParseMessage

An action of the parser.
Token

A token is an instance of a Terminal. Tokens carry post-processed terminals, as well as their position within the text file.
ModuleDescription
LALRParser

Functions to syntactically parse a series of tokens using the LALR algorithm.
Tokenizer

Functions to tokenize CharStreams.

Farkle.PostProcessor Namespace

TypeDescription
Fuser

This type contains the logic to "fuse" the multiple symbols of a production into an arbitrary object. These symbols are either transformed by a Transformer if they are terminals, or, if they are productions, are the products of previous fusers.
FuserNotFound

An exception that gets thrown when a post-processor does not find the appropriate Fuser for a production. This means that the post-processor is not properly configured.
Transformer

This type contains the logic to transform one terminal symbol to an arbitrary object.
ModuleDescription
Fuser

Functions to create Fusers.
PostProcessor

Functions to create PostProcessors, as well as some ready to use.
Transformer

Functions to create Transformers.
