Farkle.Grammar Namespace

TypeDescription
AdvanceMode

A type indicating how a group advances.
CharSet

A set of characters. See RangeSet too.
DFA

A DFA state. It defines the logic that produces Tokens out of strings. It consists of edges that the tokenizer follows, depending on the character it encounters.
EGTReadError

What can go wrong with reading an EGT file.
EndingMode

A type indicating how the ending symbol of a group is handled.
GOLDGrammar

A structure that describes a grammar - the logic under which a string is parsed. Its constructor is private; use functions like these from the EGT module to create one.
Group

A structure that describes a lexical group. In GOLD, lexical groups are used for situations where a number of recognized tokens should be organized into a single "group". This mechanism is most commonly used to handle line and block comments. However, it is not limited to "noise", but can be used for any content.
LALR
LALRAction

An action to be taken by the parser.
Production

The basic building block of a grammar's syntax. It consists of a single nonterminal called the "head". The head is defined to consist of multiple symbols making up the production's "handle".
Properties

Arbitrary metadata a grammar has. A simple key-value collection.
Symbol

A symbol of a grammar
TableCounts

A record that stores how many of each tables exist in an EGT file. It's needed only for verifying that the grammar was successfuly read.
ModuleDescription
EGT

Functions to convert EGT files to make a grammar. This is the easiest and only public API about EGT files; there's no reason to expose more.
Group

Functions to work with Groups.
Symbol

Functions to work with Symbols.

Farkle.Parser Namespace

TypeDescription
AST<'TSymbol, 'TProduction>

An Abstract Syntax Tree that describes the output of a parser. The types describing terminals and productions are arbitrary.
GOLDParser

A reusable parser created for a specific grammar that can parse input from multiple sources.
GOLDParserConfig

A set of settings to customize a parser.
LALRParser

A LALR parser. It takes a Token, and gives an LALRResult. It is a stateful operation; the type of this state is abstracted from the rest of the parser.
LALRResult
ParseInternalError

An internal error. These errors are known errors a program might experience. They could occur by manipulating the parser internal state, which is impossible from the public API. Another possible way of manifestation is creating deliberately faulty Grammars.
ParseMessage

A message from a Parser, and the position it was encountered.
ParseMessageType

A message describing what a Parser did, or what error it encountered. Some of these will only appear while using the GOLDParser API.
Parser

A type signifying the state of a parser. The parsing process can be continued by evaluating the lazy Parser values. This type is the lowest-level public API. It is modeled after the Designing with Capabilities presentation.
Reduction

A reduction will contain the tokens which correspond to the symbols of a Production. Since a reduction contains the terminals of a rule as well as the nonterminals (reductions made earlier), the parser engine creates a "parse tree" which contains a break down of the source text along the grammar's rules
Token

A token is an instance of a Symbol. Tokens carry parsed data, as well as their position within the text file.
Tokenizer

A tokenizer. What is it actually? An infinite stream of tokens (and some other information). It is infinite because the architecture currently requires it. But after one point, the tokenizer returns just EOFs forever.
TokenizerFeedback

The feedback from the tokenizer after a token is read. Because the tokenizer is completely isolated, it needs to provide some more information to the rest of the code than just the token. That's the reason of this type.
ModuleDescription
AST

Functions to work with ASTs.
Reduction

Functions to work with Reductions.

Farkle.PostProcessor Namespace

TypeDescription
PostProcessError

An error in the post-processor.
TerminalPostProcessor<'TSymbol>

This type contains the logic to transform all terminal symbols of a grammar to arbitrary objects.
Transformer

This type contains the logic to transform one terminal symbol to an arbitrary object.
UnknownTerminal

This special type signifies that a terminal symbol was not recognized by the terminal post-processor. The terminal post-processor is allowed to fail. Some symbols like the semicolons in programming languages carry no significant information up to the higher levels of the parser.
